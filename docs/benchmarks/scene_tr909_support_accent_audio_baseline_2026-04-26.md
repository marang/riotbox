# Scene TR-909 Support Accent Audio Baseline 2026-04-26

- Timestamp: `2026-04-26`
- Benchmark family: `audio-buffer-regression`
- Related ticket: `RIOTBOX-236`
- Previous baseline: `scene_tr909_support_context_baseline_2026-04-26.md`

## Scope

This baseline records the first bounded audible Scene-target TR-909 support accent.

It covers the current buffer-regression contract:

- `scene_target` source-support context gets a small support accent
- `transport_bar` remains the fallback behavior
- the accent is bounded, not a full transition engine

## Current Audio Contract

The accent is applied only when all of these are true:

- TR-909 render mode is `source_support`
- realtime source-support context is `scene_target`
- the existing drum-bus support path is active

The accent currently changes:

- trigger envelope: a small additional context boost
- render gain: a small context gain multiplier

The accent does not change:

- Scene selection policy
- action queue semantics
- transport timing
- pattern adoption
- phrase variation
- `transport_bar` fallback behavior

## Regression Coverage

Current automated coverage:

- `scene_target_context_adds_bounded_support_accent`
  - compares the same TR-909 support profile with `transport_bar` and `scene_target`
  - requires the `scene_target` peak to be higher
  - requires the peak increase to stay bounded below `1.3x`
  - requires the active sample count to stay equal
- `fixture_backed_tr909_audio_regressions_hold`
  - keeps existing TR-909 audio fixture metrics passing with context-aware render state

## Current Limits

- This is buffer-level audio QA, not a formal listening-pack approval.
- No offline WAV review pack is generated by this baseline.
- No real-session manual listening gate was run for this baseline.
- The accent is intentionally subtle so it does not pretend Riotbox has a finished Scene transition engine.

## Pass Criteria

- A `scene_target` support render is measurably louder than the same `transport_bar` render.
- The accent stays bounded and does not change trigger density.
- Existing fixture-backed TR-909 audio metrics remain within their current ranges.
- The implementation remains inside the existing realtime TR-909 render seam.
